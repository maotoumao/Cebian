Cebian 的数据主要存在浏览器本地。换设备、重装浏览器，或者想在折腾配置前留一份保险时，可以先做一次备份。

备份入口在「设置 → 备份与恢复」。这里有两种方式：下载到本地，或者上传到 WebDAV。

![截图](./snapshot.png)

## 本地备份

本地备份会把数据导出成一个 `.zip` 文件，由你自己保存。

具体操作如下：

1. 打开「设置 → 备份与恢复」
2. 在「本地备份」里点击「创建备份」
3. 选择备份模式，必要时开启「加密备份」
4. 确认后，浏览器会下载一个 `.zip` 文件

如果只是临时留一份保险，直接创建完整备份就行。

## WebDAV 备份

如果你希望多台设备之间都能恢复同一份备份，可以配置 WebDAV。

第一次使用时，需要填写：

- 服务器地址
- 用户名
- 密码或应用专用密码
- 远程目录

填好后可以先点「测试连接」。连接正常后保存，之后就可以点击「备份到 WebDAV」上传快照。

WebDAV 页面会列出服务器上已有的快照。每份快照都可以恢复，也可以删除。

## 备份模式

创建备份时有两种模式：

- 完整备份：会话、设置、技能与提示词，以及密钥都会备份
- 部分备份：自己选择要备份的内容

部分备份里可以选择：

- 会话记录
- 一并备份工作区文件
- 普通设置
- 技能与提示词
- 密钥信息

普通使用的话，建议直接用完整备份；如果只是想迁移 Prompt 或 Skill，再用部分备份。

## 加密备份

如果备份里包含「密钥信息」，强烈建议开启加密。

密钥信息包括 API Key、OAuth token、WebDAV 密码等。不开启加密的话，这些内容会跟备份文件放在一起，不适合分享给别人。

开启加密后，需要输入一个口令。恢复时也必须输入同一个口令。

> 口令不会被 Cebian 保存。请牢记口令，如果忘记口令，会导致无法恢复备份。

## 从备份恢复

本地备份和 WebDAV 快照都会进入同一个恢复流程。

恢复前，Cebian 会先显示备份里包含哪些内容。你可以选择要恢复的分类，然后选择恢复方式：

- 合并：只新增与更新，不会删除本地数据
- 替换：先清空所选分类，再按备份原样还原

不确定选哪个的话，先选「合并」。它更适合日常恢复，也不容易误删本地数据。

「替换」适合那种你明确想让当前设备变成备份里的状态的场景。比如刚装好的新浏览器，或者本地配置已经乱了，想从备份重新来一次。

## 备份内容

不同分类对应的内容大致如下：

| 分类 | 内容 |
| --- | --- |
| 会话记录 | 历史对话、消息、会话标题等 |
| 工作区文件 | 对话中生成或保存到 `/workspaces/` 的文件 |
| 普通设置 | 主题、默认模型、自定义指引、MCP 的非密钥配置等 |
| 技能与提示词 | `~/.cebian/skills/` 和 `~/.cebian/prompts/` 里的文件 |
| 密钥信息 | Provider API Key、OAuth token、MCP token、WebDAV 凭据等 |

备份不会包含浏览器 Cookie 和网站登录态。所以恢复到新设备后，有些模型订阅或网站登录可能还需要重新授权。

## Q&A

<QA>
	<QAItem q="想迁移到新设备，应该怎么备份？">创建完整备份，建议开启加密，然后在新设备上从文件或 WebDAV 恢复。</QAItem>
	<QAItem q="只想保留 Prompt / Skill 呢？">创建部分备份，只勾选「技能与提示词」即可。</QAItem>
	<QAItem q="恢复时怕覆盖本地数据怎么办？">选择「合并」。它只新增与更新，不会删除本地数据。</QAItem>
	<QAItem q="想完全按备份重来呢？">选择「替换」，但确认前最好先再导出一份当前数据。</QAItem>
	<QAItem q="WebDAV 连不上怎么办？">先检查服务器地址、用户名、密码和远程目录。有些服务需要应用专用密码。</QAItem>
</QA>
