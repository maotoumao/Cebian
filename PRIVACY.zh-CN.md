# Cebian 隐私政策

**[English](PRIVACY.md)** | 简体中文

**最近更新：** 2026-05-14

## 一句话概括

**Cebian 没有任何服务器，作者看不到你的任何数据。**

Cebian 是一个开源（[AGPL-3.0](./LICENSE)）的 Chrome 扩展，完全在你自己的浏览器里运行。你输入的每一句话、它读到的每一个页面、你配置的每一把 API Key，全部留在你本地。Cebian 唯一发起的网络请求，是**直接从你的浏览器发到你自己配置的 AI 服务商**——走 HTTPS，用你自己的 API Key 或 OAuth token 鉴权。中间没有任何由 Cebian 作者运营的服务器。

卸载扩展，本地数据随之消失。

---

## 一、适用范围

本政策说明 Cebian 浏览器扩展（下称「Cebian」「本扩展」或「我们」）如何处理用户数据。

本政策**不**覆盖以下内容：

- 你配置的 AI 服务商（OpenAI、Anthropic、Google 等）的隐私实践——这些由它们各自的隐私政策约束
- 你选择接入的任何 MCP（Model Context Protocol）服务的隐私实践
- 任何由你或第三方修改、二次打包或自行部署的 Cebian 衍生版本

源码仓库：<https://github.com/maotoumao/Cebian>

---

## 二、我们处理的数据

Cebian 只在**你主动触发对应功能时**才会处理下表中的数据，绝不在后台静默收集。

| 数据类别 | 具体内容 | 何时被读取 | 流向 |
| :-- | :-- | :-- | :-- |
| **网站内容** | 页面 URL、标题、正文、你点选的 DOM 元素 | 你向 AI 发送消息时 | 本地浏览器 → 你选择的 AI 服务商 |
| **用户活动** | 当前标签页 URL / 标题、你选中的文本，以及其他已打开标签页的 URL 与标题（便于 AI 了解你现在都开了什么） | 你向 AI 发送消息时 | 本地浏览器 → 你选择的 AI 服务商 |
| **剪贴板内容** | 仅当提示词模板使用 `{{clipboard}}` 变量时读取系统剪贴板 | 你提交此类模板时 | 本地浏览器 → 你选择的 AI 服务商 |
| **浏览历史** | 通过 `chrome.history` 读取 | 仅当你明确要求 AI 查询历史时 | 本地浏览器 → 你选择的 AI 服务商 |
| **书签** | 通过 `chrome.bookmarks` 读写 | 仅当你明确要求 AI 查看或修改书签时 | 本地浏览器 → 你选择的 AI 服务商 |
| **Cookie** | 通过 `chrome.cookies` 读取 | 仅当你明确要求 AI 检查某个站点的 Cookie 时 | 本地浏览器 → 你选择的 AI 服务商 |
| **常用网站** | 通过 `chrome.topSites` 读取 | 仅当你明确要求 AI 列出时 | 本地浏览器 → 你选择的 AI 服务商 |
| **最近关闭的标签 / 会话** | 通过 `chrome.sessions` 读取 | 仅当你明确要求 AI 读取时 | 本地浏览器 → 你选择的 AI 服务商 |
| **下载记录** | 通过 `chrome.downloads` 读取 | 仅当你明确要求 AI 查询下载时 | 本地浏览器 → 你选择的 AI 服务商 |
| **录制的交互会话** | 点击、按键（包括 Enter / Backspace 等特殊键）、输入的文本、滚动、页面导航、以及 DOM 元素选择器 | 仅当你从 Cebian 界面主动启动录制时 | 录制过程中保在内存；停止后作为附件带到聊天消息中，随对应会话一同存入 IndexedDB |
| **麦克风音频（语音输入）** | 实时麦克风音频，仅用于把语音转写为聊天输入框中的文字 | 仅当你从输入框的麦克风按钮主动启动语音输入时 | **音频**完全在**本地设备**上由浏览器内置语音引擎处理，**绝不离开你的设备**——不会发送到任何服务器，包括 AI 服务商。转写出的**文字**会出现在输入框供你确认；默认**不发送到任何地方**。若你启用 AI 修正（可选功能），该段转写文字会像普通消息一样发送给你配置的 AI 服务商以做修正——音频本身仍绝不离开你的设备 |
| **API Key 与 OAuth Token** | 你为各 AI 服务商填写的凭证 | 你在「设置」中输入或更新时 | 本地存储于 `chrome.storage.local`，仅作为 `Authorization` 头发给对应服务商的 API |
| **MCP 服务器配置** | 你添加的任意 Model Context Protocol 服务器的 URL、自定义请求头、Bearer Token | 你在「设置」中添加或编辑时 | 本地存储于 `chrome.storage.local`，仅发给你配置的那个 MCP 服务器 |
| **聊天记录、设置** | 会话、个人偏好 | 你使用扩展过程中持续保存 | 本地存储于 IndexedDB（`cebian` 数据库）与 `chrome.storage.local` |
| **提示词模板、技能、虚拟文件系统** | 你编写或导入的提示词模板、技能包；在扩展内虚拟文件系统中创建的文件 | 你使用扩展过程中持续保存 | 本地存储于另一个 IndexedDB 数据库（`cebian-vfs`） |

我们**不会**收集或处理：你的姓名、邮箱、账号信息、支付信息、IP 地址、设备标识、浏览器指纹，以及任何统计或埋点数据。

## 三、数据流向

```
┌─────────────────────────────────────────┐
│  你的浏览器（仅本地）                   │
│                                         │
│   ┌───────────────────────────────┐     │
│   │  Cebian 扩展                  │     │
│   │  • IndexedDB（聊天、技能）    │     │
│   │  • chrome.storage（设置、     │     │
│   │    API Key）                  │     │
│   └────────────────┬──────────────┘     │
│                    │                    │
│                    │ 你发送一条消息     │
│                    ▼                    │
│   ┌───────────────────────────────┐     │
│   │  读取相关上下文               │     │
│   │  （页面文本、选区等）         │     │
│   └────────────────┬──────────────┘     │
└────────────────────┼────────────────────┘
                     │ HTTPS（用你的 API Key）
                     ▼
┌─────────────────────────────────────────┐
│  你自己配置的 AI 服务商                 │
│  （OpenAI / Anthropic / Google / xAI /  │
│   Groq / OpenRouter / Mistral / 你自己  │
│   的 OpenAI 兼容端点 / ……）             │
└─────────────────────────────────────────┘
```

Cebian 作者**不运行任何服务器**。没有代理、没有埋点端点、没有错误上报、没有遥测心跳。

唯一的例外也透明披露：当你打开**「设置 → 关于」**页面时，扩展会从 `https://api.github.com/repos/maotoumao/Cebian/releases/latest` 拉取最新发布版本号，告诉你是否有更新。结果会缓存 6 小时。该请求**不携带任何用户数据**，但与任何 HTTP 请求一样，GitHub 可能会记录请求方的 IP 地址与 User-Agent。扩展本体的二进制更新由 Chrome 商店负责，不由我们分发。

## 四、我们绝不做的事

- ❌ 我们不运行任何接收用户数据的后端服务
- ❌ 我们不收集任何统计、埋点、崩溃报告或使用数据
- ❌ 我们不展示任何广告
- ❌ 我们不会向你自己配置的 AI 服务商之外的任何第三方出售、出租、共享或转让你的数据
- ❌ 我们不会将你的数据用于训练任何模型
- ❌ 我们不要求你注册或登录任何账号
- ❌ 你不主动发消息，我们就不会读取或传输页面内容

## 五、第三方 AI 服务商

你发送消息时，Cebian 直接从你的浏览器把请求发到你选择的服务商 API。Cebian 目前支持（不限于）：

- OpenAI、Anthropic、Google Gemini、xAI、Groq、OpenRouter、DeepSeek、Mistral、MiniMax、MiniMax（国内版）、Kimi、zAI
- GitHub Copilot、OpenAI Codex（OAuth 登录）
- 你自己添加的任意 OpenAI 兼容端点（包括自部署模型）

数据离开浏览器后，由对应服务商的隐私政策约束。它们是否记录、保留、或将你的输入用于训练，**完全由它们的条款决定**——Cebian 对此既没有可见性也没有控制权。请在启用任何服务商前查阅它们的隐私政策。

如果你通过 MCP（Model Context Protocol）接入了外部工具，对那些端点同样适用上述说明。

## 六、Chrome 权限说明

Cebian 在 manifest 中申请以下权限，每一项都仅用于下方描述的用途。

| 权限 | Cebian 为何需要 |
| :-- | :-- |
| `sidePanel` | 在浏览器侧边栏渲染 Cebian 界面 |
| `activeTab` | 读取你当前正在看的标签页的 URL、标题与内容，让 AI 能基于它回答问题 |
| `tabs` | 列出已打开的标签页，让 AI 知道你都开了些什么、并按你的要求切换 |
| `scripting` | 向当前标签页注入脚本以读取页面文本、点选 DOM 元素，并在你许可后执行 AI 给出的 JavaScript 片段 |
| `storage` | 把你的设置、提示词模板、技能、API Key 持久化到 `chrome.storage.local` |
| `alarms` | 周期性（每 30 分钟）刷新 OAuth Token，让 GitHub Copilot 等登录态的服务商保持有效 |
| `offscreen` | 承载 Service Worker 无法提供的 DOM/音频上下文（例如剪贴板、音频处理） |
| `debugger` | 通过 Chrome DevTools Protocol 提供高级页面交互能力（移动端模拟、截图、网络抓包等），仅作用于你正在操作的标签页 |
| `webNavigation` | 监听标签页的导航事件，让上下文（URL、标题）始终与你正在看的页面一致 |
| `bookmarks` | **仅当你明确要求 AI 管理书签时**读写你的书签 |
| `history` | **仅当你明确要求 AI 查询历史时**读取浏览历史 |
| `cookies` | **仅当你明确要求 AI 检查某个站点 Cookie 时**读取（例如调试用途） |
| `topSites` | **仅当你明确要求 AI 列出时**读取常用网站 |
| `sessions` | **仅当你明确要求 AI 读取时**获取最近关闭的标签或会话 |
| `downloads` | **仅当你明确要求 AI 操作时**查询或管理下载 |
| `notifications` | 为你委托给 AI 的长时间任务展示桌面通知 |
| `clipboardRead` | **仅当你提交带 `{{clipboard}}` 变量的提示词模板时**读取系统剪贴板 |
| `host_permissions: <all_urls>` | AI 可能被你问到**任何**网站的相关问题；没有全 URL 主机权限，Cebian 就无法在任意站点上读取或协作。该权限只用于读取/操作你正在使用的标签页，绝不在后台扫描站点 |

## 七、数据存储与删除

Cebian 产生的所有数据都存在你本地浏览器，具体位置：

- **`chrome.storage.local`**——设置、API Key、OAuth Token、MCP 服务器配置（包含 Bearer Token）
- **IndexedDB — `cebian` 数据库**——聊天会话与消息历史（包括你附加到聊天中的交互录制）
- **IndexedDB — `cebian-vfs` 数据库**——提示词模板、技能包，以及你在 Cebian 虚拟文件系统中创建或导入的文件

你可以随时删除这些数据：

- **删除单条聊天**：打开历史面板，删除对应会话
- **删除全部数据**：在 `chrome://extensions` 中卸载 Cebian，Chrome 会一并清除该扩展的 `chrome.storage` 与 IndexedDB 数据

由于作者不运营任何服务器，**不存在任何远程副本**需要你来要求删除。

## 八、未成年人隐私

Cebian 并非面向 13 岁以下儿童的产品。我们不会在知情的情况下收集儿童的个人信息。

---

## 九、安全

- API Key 与 OAuth Token 存储于 `chrome.storage.local`，由 Chrome 按扩展沙箱隔离
- 所有发往 AI 服务商的网络请求均走 HTTPS
- Cebian 自身不会把凭证传到对应服务商 API 的 `Authorization` 头之外的任何地方

但请注意：能物理接触到你已解锁电脑用户资料的人，依然能读到 `chrome.storage.local`。请像对待保存了 AI 服务商密钥的任何其他设备一样对待这台机器。
